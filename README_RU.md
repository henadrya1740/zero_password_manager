# 🚀 Почему Zero Password Manager?

## ☁️ Никакого облака — по-настоящему, а не «почти»

Большинство менеджеров паролей, заявляющих «zero-knowledge», всё равно хранят ваши зашифрованные данные **на своих серверах**. Это значит:
- Их могут обязать раскрыть данные по решению суда
- Они могут быть взломаны
- Они могут закрыться (и забрать ваши данные)
- Они могут изменить политику конфиденциальности в любой момент

**Zero Password Manager идёт дальше:**

> Ваше зашифрованное хранилище живёт на **вашей машине**. Точка.

✔ Ни Google Cloud, ни AWS, ни Azure — никакого стороннего хранилища
✔ Никакого аккаунта на чужом сайте
✔ Никакой ежемесячной подписки, которая растёт каждый год
✔ Никаких писем «нас взломали, смените пароли»
✔ Нет риска отключения сервиса — это ваш сервер
✔ Работает полностью офлайн в локальной сети

Вот как выглядит **настоящее владение данными**.

## 🔐 Шифрование военного уровня — на ваших условиях

Каждый пароль шифруется **до того, как покидает ваше устройство**:

- **AES-256-GCM** — тот самый шифр, который используют армии и финансовые институты по всему миру
- **Argon2id** для получения ключа — золотой стандарт хеширования паролей, устойчивый к атакам на GPU и ASIC (3 итерации, 64 МБ памяти)
- **12-байтовый случайный nonce** при каждом шифровании — гарантирует уникальность, даже если вы сохраняете один и тот же пароль дважды
- Сервер хранит **только зашифрованные блобы** — он буквально не может прочитать ваши пароли, даже если захочет
- Мастер-пароль **никогда не передаётся по сети** — вообще

```
Ваше устройство → ключ из мастер-пароля (Argon2id) → шифрование → блоб → сервер
Сервер имеет: только зашифрованный блоб. Ключа нет. Открытого текста нет. Zero Knowledge.
```

## 🛡️ Усиленная 2FA — обязательная, а не дополнительная

2FA **обязательна с первого дня**, а не опциональная надстройка:

- **TOTP** поддержка (Google Authenticator, Aegis, Microsoft Authenticator, любое совместимое приложение)
- **QR-код** при регистрации — отсканировал и готово
- **Защита на уровне операций** — можно требовать свежий OTP-код для каждого чтения хранилища, каждой записи или просмотра аудит-лога (настраивается)
- **Защита от повторных атак** — каждый таймкод можно использовать только один раз, даже в пределах допустимого окна
- **Rate limiting** — 5 попыток в минуту с принудительной задержкой при неверном коде

## 📁 Папки паролей — наведите порядок в хранилище

По мере роста хранилища поиск нужного пароля не должен превращаться в скролл бесконечного списка.

**Папки позволяют группировать пароли так, как удобно вам:**

- 🏠 `Дом` — WiFi, роутер, умный дом
- 💼 `Работа` — рабочие инструменты, VPN, внутренние системы
- 🏦 `Финансы` — банки, инвестиции, крипто-кошельки
- 🎮 `Игры` — Steam, Epic, консольные аккаунты
- ☁️ `Облако` — AWS, хостинг-панели, CI/CD
- ...или любая структура, которая имеет смысл именно для вас

**Как это работает:**
- Создайте папку с **именем**, выберите из **12 цветов** и **16 иконок**
- **Панель папок** на главном экране фильтрует хранилище в один тап
- Выбор папки при добавлении/редактировании записи — **всегда опционально**
- Удаление папки **никогда не удаляет пароли** — они просто становятся без папки
- Все данные папок привязаны к пользователю на сервере — никакой утечки между аккаунтами

## 🔒 Биометрическая разблокировка

Не вводите мастер-пароль каждый раз:

- Поддержка **отпечатка пальца** и **Face ID**
- **PIN-код** как резервный способ для устройств без биометрии
- Биометрические данные остаются на устройстве — **никогда не покидают его**
- Автоблокировка после настраиваемого таймаута неактивности

## 📜 Аудит-лог — знайте, что происходит в вашем хранилище

Каждое важное действие записывается:

- Попытки входа (с IP-адресом)
- Чтение и запись хранилища
- Создание, обновление и удаление паролей
- Включение/выключение 2FA
- Полная история с временными метками

Вы видите всё, что когда-либо происходило в вашем хранилище. Никаких чёрных ящиков.

# 🛡 Архитектура безопасности

```
┌─────────────────────────────────────────────────┐
│                 Ваше устройство                  │
│  Мастер-пароль → Argon2id → 256-битный ключ     │
│  Открытый текст → AES-256-GCM → Зашифрованный  │
└────────────────────┬────────────────────────────┘
                     │ HTTPS (только зашифрованный блоб)
┌────────────────────▼────────────────────────────┐
│                 Ваш сервер                       │
│  Хранит: зашифрованный блоб, хеш логина, соль  │
│  Не может прочесть: пароли, заметки, сид-фразы │
│  JWT-аутентификация + rate limiting + аудит-лог │
└─────────────────────────────────────────────────┘
```

**Стек безопасности:**
- `Argon2id` — хеширование паролей (time_cost=3, memory=64MB, parallelism=1)
- `AES-256-GCM` — аутентифицированное шифрование со случайными nonce
- `HS256 JWT` — краткосрочные access-токены (15 мин) + refresh-токены (7 дней)
- `slowapi` — rate limiting на всех чувствительных эндпоинтах
- `HSTS + CSP + X-Frame-Options` — HTTP security headers

# 📦 Self-Hosting за 5 минут

## 🐍 1. Запуск бэкенда

```bash
cd server
pip install -r requirements.txt
cp env.example .env
# Отредактируйте .env — задайте JWT_SECRET_KEY и ALLOWED_ORIGINS
python -m uvicorn main:app --host 0.0.0.0 --port 3000
```

Сгенерируйте надёжный JWT-ключ:
```bash
python -c "import secrets; print(secrets.token_hex(32))"
```

## 📱 2. Запуск Flutter-приложения

```bash
flutter pub get
# Отредактируйте env.dev — укажите IP вашего сервера в API_BASE_URL
flutter run
```

## 📦 3. Сборка для мобильных устройств

```bash
# Android
flutter build apk --release

# iOS (требует macOS и Xcode)
flutter build ios --release
```
